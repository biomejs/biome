You can use Biome with a pre-commit tool.
This can automatically format, lint, or check your staged files when running `git commit`.

## Lefthook

Install [`lefthook`](https://github.com/evilmartians/lefthook/blob/master/docs/install.md#-nodejs), and use of the following examples.

### Format and lint all files

This snippet will format and lint all supported files in the repository, and add them to the commit.

```yaml
pre-commit:
  commands:
    check:
      glob: "*.{js,ts,jsx,tsx,json}"
      run: biome check --apply . && git add .
```

### Format and lint staged files only

This snippet will format and lint only the staged files, and add them to the commit.

```yaml
pre-commit:
  commands:
    check:
      glob: "*.{js,ts,jsx,tsx,json}"
      run: biome check --apply {staged_files} && git add {staged_files}
```

## `lint-staged`

Install [`lint-staged`](https://www.npmjs.com/package/lint-staged#installation-and-setup) and open its configuration file/section and update it with the needs you require. Remember to use the CLI option `--no-errors-on-unmatched` in your command, to silent possible errors in case *no files are processed*.

Here's some example of commands that you could find useful when running the git hooks:

```json5
{
  "lint-staged": {
    // Run Biome to files that have the following extensions: js, ts, jsx, tsx, json and jsonc
    "**.{js|ts|jsx|tsx|json|jsonc}": [
      "biome format --write --no-errors-on-unmatched", // format and write files
      "biome lint --apply --no-errors-on-unmatched", // apply safe fixes when linting files
      "biome check --apply --no-errors-on-unmatched", // format, sort imports and apply safe fixes when checking files
      "biome check --apply --organize-imports-enabled=false --no-errors-on-unmatched", // format and apply safe fixes when checking files
      "biome check --apply-unsafe --no-errors-on-unmatched", // format, sort imports, apply safe fixes and apply unsafe fixes when checking files
    ]
  }
}
```

## pre-commit

Biome provides four [pre-commit](https://pre-commit.com/) hooks via the [biomejs/pre-commit](https://github.com/biomejs/pre-commit) repository.

| hook `id`       | description                                                                 |
| --------------- | --------------------------------------------------------------------------- |
| `biome-ci`      | Check formatting, check if imports are organized, and lints                 |
| `biome-check`   | Format, organize imports, lint, and apply safe fixes to the committed files |
| `biome-format`  | Format the committed files                                                  |
| `biome-lint`    | Lint and apply safe fixes to the committed files                            |

In the following example, we assume that you [installed pre-commit](https://pre-commit.com/index.html#install) and run `pre-commit install` in your repository.
if you want to use the `biome-check` hook, add the following pre-commit configuration to the root of your project in a file named `.pre-commit-config.yaml`:

```yaml
repos:
-   repo: https://github.com/biomejs/pre-commit
    rev: "0.1.0"  # Use the sha / tag you want to point at
    hooks:
    -   id: biome-check
        additional_dependencies: ["@biomejs/biome@1.4.1"]
```

This will run `biome check --apply` when you run `git commit`.

Note that you must specify which version of Biome to use thanks to the `additional_dependencies` option.
[pre-commit](https://pre-commit.com/) separately installs tools and need to know which one to install.

If Biome is already installed as a `npm` package in your local repository,
then it can be a burden to update both `package.json` and `.pre-commit-config.yaml` when you update Biome.
Instead of using the provided Biome hooks, you can specify your own [local hook](https://pre-commit.com/index.html#repository-local-hooks).

For example, if you use `npm`, you can write the following hook in `.pre-commit-config.yaml`:

```yaml
repos:
  - repo: local
    hooks:
      - id: local-biome-check
        name: biome check
        entry: npx biome check --apply --files-ignore-unknown=true --no-errors-on-unmatched
        language: system
        types: [text]
        files: "\\.(jsx?|tsx?|c(js|ts)|m(js|ts)|d\\.(ts|cts|mts)|jsonc?)$"
```

The pre-commit option `files` is optional,
because Biome is able to ignore unknown files (using the option `--files-ignore-unknown=true`).

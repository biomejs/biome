## 通过 CLI 使用语法检查器

您可以通过运行 CLI 来检查可能的错误，使用以下命令：

```shell
biome lint ./src
```

有关所有可用选项的更多信息，请查看 [CLI 页面](/reference/cli#biome-lint)

## 规则支柱

我们认为规则应该具有信息性，并向用户解释为什么会触发规则，并告诉用户应该如何修复错误。规则应遵循以下**支柱**：
1. 向用户解释错误。通常，这是诊断的消息。
2. 向用户解释**为什么**触发错误。通常，这是通过附加节点实现的。
3. 告诉用户他们应该做什么。通常，这是通过代码操作实现的。如果代码操作不适用，注释应告诉用户应该如何修复错误。

如果您认为规则没有遵循这些支柱，请[提交问题](https://github.com/biomejs/biome/issues/new?assignees=&labels=S-To+triage&projects=&template=01_bug.yml&title=%F0%9F%90%9B+%3CTITLE%3E)。

## 代码修复

语法检查规则可能提供自动代码修复。Biome 区分了两种类型的修复。

### 安全修复

安全修复保证不会改变您的代码的语义。他们可以在没有明确审查的情况下应用。

要应用 _安全修复_，使用 `--apply`：

```shell
biome check --apply ./src
```

### 不安全修复

不安全修复可能会改变您的程序的语义。因此，建议手动审查更改。

要应用 _不安全修复_，使用 `--apply-unsafe`：

```shell
biome check --apply-unsafe ./src
```

## 忽略代码

有时，开发人员希望忽略特定代码行的语法检查规则。
您可以通过在发出语法检查诊断的行上方添加抑制注释来实现此目的。

抑制注释的格式如下：

```js
// biome-ignore lint: <explanation>
// biome-ignore lint/suspicious/noDebugger: <explanation>
```

其中

- `biome-ignore` 是抑制注释的开始；
- `lint` 抑制语法检查器；
- `/suspicious/noDebugger`：**可选**，您要抑制的规则的组和名称；
- `<explanation>` 解释为什么禁用规则

这是一个例子：

```ts
// biome-ignore lint: reason
debugger;
// biome-ignore lint/suspicious/noDebugger: reason
debugger;
```


## 配置

### 启用语法检查规则

推荐的规则默认启用，并发出错误严重性的诊断。
默认情况下禁用不推荐的规则，但可以通过配置启用它们。
这些规则发出的诊断在文档中以警告严重性显示。

要启用规则，您需要根据您的需要更改其诊断严重性：

```json title="biome.json"
{
  "linter": {
    "enabled": true,
    "rules": {
      "style": {
        "useBlockStatements": "error",
        "useShorthandArrayType": "error",
        "noShoutyConstants": "warn"
      }
    }
  }
}
```

### 禁用语法检查规则

只需在其配置中添加 `"off"` 作为值。例如：

```json title="biome.json"
{
  "linter": {
    "enabled": true,
    "rules": {
      "suspicious": {
        "noCommentText": "off"
      },
      "style": {
        "noUnusedTemplateLiteral": "off"
      }
    }
  }
}
```

### 更改诊断严重性

Biome 的大多数规则都会发出**错误**，但您可以自由更改其严重性。
只需将 `"warn"` 添加为规则的值。示例：

```json title="biome.json"
{
  "linter": {
    "enabled": true,
    "rules": {
      "suspicious": {
        "noCommentText": "warn"
      }
    }
  }
}
```

在进行重构并需要使 CI 通过的情况下，这是有用的。

## 规则选项

一些规则有选项。
当他们确实接受一些选项时，您可以通过改变规则的值的形状来传递它们。

```json title="biome.json"
{
  "linter": {
    "enabled": true,
    "rules": {
      "correctness": {
        "noCommentText": {
          "level": "warn",
          "options": {}
        }
      }
    }
  }
}
```

- `level` 将指示诊断的严重性，有效值为：`"off"`、`"warn"` 和 `"error"`；
- `options` 将根据规则改变。

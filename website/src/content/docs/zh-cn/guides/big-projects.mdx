---
title: 在大型项目中使用 Biome
description: 如何在大型项目中设置 Biome 的小指南
---

Biome 可以提供一些工具，帮助你在大型项目中正确使用它，例如包含多个项目的单仓库或工作区。


## 使用多个配置文件

当你使用 Biome 的功能 - 无论是使用 CLI 还是 LSP - 该工具会使用当前工作目录查找最近的配置文件。

如果 Biome 在那里找不到配置文件，它会**开始向上**遍历文件系统的目录，直到找到一个。

你可以利用这个功能，根据项目/文件夹应用不同的设置。

假设我们有一个项目，包含一个后端应用和新的前端应用。


```
app
├── backend
│   ├── biome.json
│   └── package.json
└── frontend
    ├── biome.json
    ├── legacy-app
    │   └── package.json
    └── new-app
        └── package.json
```

这意味着当你从文件 `app/backend/package.json` 运行脚本时，Biome 将使用配置文件 `app/backend/biome.json`。

当你从 `app/frontend/legacy-app/package.json` 或 `app/frontend/new-app/package.json` 运行脚本时，Biome 将使用配置文件 `app/frontend/biome.json`。

## 分享配置

可以使用 [`extends`](/reference/configuration#extends) 配置选项在文件之间分解选项。

假设我们有以下需求：
- `legacy-app` 必须使用空格进行格式化；
- `backend` 和 `new-app` 必须使用制表符进行格式化；
- 所有应用必须使用 120 的行宽进行格式化；
- `backend` 应用需要一些额外的 linting；

我们首先在 `app/biome.json` 创建一个新的配置文件，并在那里放置共享选项：


```json title="app/biome.json"
{
  "formatter": {
    "enabled": true,
    "lineWidth": 120
  }
}
```

现在让我们**移动** `app/frontend/biome.json` 到 `app/frontend/legacy-app/`，因为那是我们需要使用不同格式化的地方。


```json title="app/frontend/legacy-app/biome.json"
{
  "formatter": {
    "indentStyle": "space"
  }
}
```

然后，我们告诉 Biome 使用 `extends` 属性从主 `app/biome.json` 文件继承所有选项：

```json title="app/frontend/legacy-app/biome.json" ins={2}
{
   "extends": ["../../biome.json"],
   "formatter": {
     "indentStyle": "space"
   }
}
```

让我们跳到 `app/backend/biome.json`，在那里我们需要启用 linting：


```json title="app/backend/biome.json"
{
  "extends": ["../biome.json"],
    "linter": {
      "enabled": "true",
      "rules": {
        "recommended": true
      }
   }
}
```

## 不使用 Node.js 使用 Biome

如果您还没有使用 Node 或 npm（或任何其他包管理器），那么使用 Biome 的独立 CLI 二进制文件可能是一个很好的选择。
换句话说，Biome 不应该是您拥有 `package.json` 的唯一原因。

:::note
如果您已经使用 npm 或其他包管理器，那么使用包管理器是安装 Biome 的[首选方式](/guides/getting-started#installation)。
您已经熟悉了工具，安装和更新更简单。
:::

### Homebrew

Biome 可以作为 macOS 和 Linux 用户的 [Homebrew formula](https://formulae.brew.sh/formula/biome)。

```shell
brew install biome
```

## 系统要求

- Windows（包括 WSL）、macOS 或 Linux
- x86_64 或 ARM64

## 支持的平台

您必须为 Biome 选择正确的平台二进制文件。以下表格应该可以帮助您做到这一点。

| CPU 架构 | Windows       | macOS                        | Linux         | Linux (musl)       |
| ---------------- | ------------- | ---------------------------- | ------------- | ------------------ |
| `arm64`          | `win32-arm64` | `darwin-arm64` (M1 或更新) | `linux-arm64` | `linux-arm64-musl` |
| `x64`            | `win32-x64`   | `darwin-x64`                 | `linux-x64`   | `linux-x64-musl`   |

:::note
对于 Windows Subsystem for Linux (WSL)，请使用 Linux 变体。
:::

## 安装 Biome

要安装 Biome，从 GitHub 上的 [最新 CLI 发布](https://github.com/biomejs/biome/releases)中获取适用于您平台的可执行文件，并赋予其执行权限。

```shell
# macOS arm (M1 或更新)
curl -L https://github.com/biomejs/biome/releases/download/cli%2Fv<version>/biome-darwin-arm64 -o biome
chmod +x biome

# Linux (x86_64)
curl -L https://github.com/biomejs/biome/releases/download/cli%2Fv<version>/biome-linux-x64 -o biome
chmod +x biome

# Windows (x86_64, Powershell)
Invoke-WebRequest -Uri "https://github.com/biomejs/biome/releases/download/cli%2Fv<version>/biome-win32-x64.exe" -OutFile "biome.exe"
```

:::note
请确保将 `<version>` 替换为您要安装的 Biome 版本。
:::

现在，您可以通过简单地运行 `./biome` 来使用 Biome。

## biome

Biome官方CLI。用于检查项目的健康状况或运行以检查单个文件。

**用法**: **`biome`** _`COMMAND ...`_

**可用选项:**

- **`-h`**, **`--help`** &mdash;
  打印帮助信息
- **`-V`**, **`--version`** &mdash;
  打印版本信息

**可用命令:**

- **`version`** &mdash;
  显示Biome版本信息并退出
- **`rage`** &mdash;
  打印用于调试的信息
- **`start`** &mdash;
  启动Biome守护进程服务器进程
- **`stop`** &mdash;
  停止Biome守护进程服务器进程
- **`check`** &mdash;
  对请求的文件运行格式化器、语法检查器和导入排序。
- **`lint`** &mdash;
  对一组文件运行各种检查。
- **`format`** &mdash;
  对一组文件运行格式化器。
- **`ci`** &mdash;
  在CI环境中使用的命令。对请求的文件运行格式化器、语法检查器和导入排序。
- **`init`** &mdash;
  引导新的Biome项目。创建带有一些默认值的配置文件。
- **`lsp-proxy`** &mdash;
  充当通过stdin/stdout的语言服务器协议的服务器
- **`migrate`** &mdash;
  在有破坏性更改时更新配置

## biome version

显示Biome版本信息并退出

**用法**: **`biome`** **`version`**

**应用于所有命令的全局选项**

- **`    --colors`**=_`<off|force>`_ &mdash;
  设置标记的格式化模式: "off"将所有内容打印为纯文本，"force"强制使用ANSI格式化标记，即使判断控制台输出不兼容
- **`    --use-server`** &mdash;
  连接到运行中的Biome守护进程服务器实例。
- **`    --verbose`** &mdash;
  在诊断上打印附加详细建议
- **`    --config-path`**=_`PATH`_ &mdash;
  设置文件系统路径为biome.json配置文件的目录
- **`    --max-diagnostics`**=_`NUMBER`_ &mdash;
  限制显示的诊断数量。
  [默认值: 20]
- **`    --skip-errors`** &mdash;
  跳过包含语法错误的文件，而不是发出错误诊断。
- **`    --no-errors-on-unmatched`** &mdash;
  隐藏在执行命令期间未处理任何文件的情况下发出的错误。
- **`    --error-on-warnings`** &mdash;
  告诉Biome如果某些诊断发出警告，则以错误代码退出。
- **`    --log-level`**=_`<none|debug|info|warn|error>`_ &mdash;
  日志级别。从最详细到最不详细的顺序为: debug，info，warn，error。

  值`none`不会显示任何日志。

  [默认值: none]

- **`    --log-kind`**=_`<pretty|compact|json>`_ &mdash;
  日志的外观。
  [默认值: pretty]
- **`    --diagnostic-level`**=_`<info|warn|error>`_ &mdash;
  要显示的诊断级别。从最低到最重要的顺序为: info，warn，error。传递`--diagnostic-level=error`将导致Biome仅打印仅包含错误的诊断。
  [默认值: info]

**可用选项:**

- **`-h`**, **`--help`** &mdash;
  打印帮助信息

## biome rage

打印用于调试的信息

**用法**: **`biome`** **`rage`** \[**`--daemon-logs`**\]

**适用于所有命令的全局选项**

- **`    --colors`**=_`<关闭|启用>`_ &mdash;
  设置标记的格式化模式："关闭"将所有内容以纯文本形式打印，"启用"强制使用ANSI格式化标记，即使控制台输出被确定不兼容
- **`    --use-server`** &mdash;
  连接到正在运行的Biome守护进程服务器实例
- **`    --verbose`** &mdash;
  打印有关诊断的额外详细建议
- **`    --config-path`**=_`路径`_ &mdash;
  设置biome.json配置文件所在的文件系统路径
- **`    --max-diagnostics`**=_`数字`_ &mdash;
  限制显示的诊断数量
  [默认值: 20]
- **`    --skip-errors`** &mdash;
  跳过包含语法错误的文件，而不是发出错误诊断
- **`    --no-errors-on-unmatched`** &mdash;
  在执行命令期间未处理任何文件的情况下，抑制将发出的错误
- **`    --error-on-warnings`** &mdash;
  如果某些诊断发出警告，告知Biome以错误代码退出
- **`    --log-level`**=_`<无|调试|信息|警告|错误>`_ &mdash;
  日志级别。从最详细到最简洁顺序为：调试，信息，警告，错误。

  值 `无` 将不显示任何日志。

  [默认值: 无]

- **`    --log-kind`**=_`<漂亮|紧凑|JSON>`_ &mdash;
  日志的样式。
  [默认值: 漂亮]
- **`    --diagnostic-level`**=_`<信息|警告|错误>`_ &mdash;
  显示的诊断级别。从最不重要到最重要的顺序为：信息，警告，错误。传递 `--diagnostic-level=错误` 将导致Biome仅打印只包含错误的诊断。
  [默认值: 信息]

**可用选项:**

- **`    --daemon-logs`** &mdash;
  打印Biome守护进程服务器的日志
- **`-h`**, **`--help`** &mdash;
  打印帮助信息

## biome check

对请求的文件运行格式化器、代码检查器和导入排序。

**用法**: **`biome`** **`check`** \[**`--apply`**\] \[**`--apply-unsafe`**\] \[_`路径`_\]...

**biome.json文件中包含的配置**

- **`    --vcs-client-kind`**=_`<git>`_ &mdash;
  客户端的类型。
- **`    --vcs-enabled`**=_`<true|false>`_ &mdash;
  Biome是否应与VCS客户端集成
- **`    --vcs-use-ignore-file`**=_`<true|false>`_ &mdash;
  Biome是否应使用VCS ignore文件。当为[true]时，Biome将忽略在ignore文件中指定的文件。
- **`    --vcs-root`**=_`路径`_ &mdash;
  Biome应在其中检查VCS文件的文件夹。默认情况下，Biome将使用找到`biome.json`的相同文件夹。

  如果Biome找不到配置，它将尝试使用当前工作目录。如果找不到当前工作目录，Biome将不使用VCS集成，并发出诊断信息。

- **`    --files-max-size`**=_`NUMBER`_ &mdash;
  源代码文件的最大允许大小（以字节为单位）。超过此限制的文件将由于性能原因被忽略。默认为 1 MiB
- **`    --files-ignore-unknown`**=_`<true|false>`_ &mdash;
  告诉Biome在处理未知文件时不发出诊断信息
- **`    --indent-style`**=_`<tab|space>`_ &mdash;
  缩进样式。
- **`    --indent-size`**=_`NUMBER`_ &mdash;
  缩进的大小，默认为2（已弃用，请使用`indent-width`）
- **`    --indent-width`**=_`NUMBER`_ &mdash;
  缩进的大小，默认为2
- **`    --line-ending`**=_`<lf|crlf|cr>`_ &mdash;
  行尾的类型。
- **`    --line-width`**=_`NUMBER`_ &mdash;
  一行的最大宽度。默认为80。
- **`    --quote-style`**=_`<double|single>`_ &mdash;
  JavaScript代码中使用的引号类型。默认为双引号。
- **`    --jsx-quote-style`**=_`<double|single>`_ &mdash;
  JSX中使用的引号类型。默认为双引号。
- **`    --quote-properties`**=_`<preserve|as-needed>`_ &mdash;
  对象中的属性何时引用。默认为根据需要添加引号。
- **`    --trailing-comma`**=_`<all|es5|none>`_ &mdash;
  在多行逗号分隔的句法结构中尽可能打印尾逗号。默认为"all"。
- **`    --semicolons`**=_`<always|as-needed>`_ &mdash;
  格式化程序是否为所有语句打印分号，或仅在由于ASI而必要的for语句中打印分号。
- **`    --arrow-parentheses`**=_`<always|as-needed>`_ &mdash;
  是否在箭头函数上添加非必要的括号。默认为"always"。
- **`    --bracket-spacing`**=_`<true|false>`_ &mdash;
  是否在对象字面量的括号周围插入空格。默认为true。
- **`    --bracket-same-line`**=_`<true|false>`_ &mdash;
  是否将多行HTML/JSX标签的闭合括号与最后一行的末尾紧挨在一起，而不是单独放在下一行。默认为false。
- **`    --javascript-formatter-enabled`**=_`<true|false>`_ &mdash;
  控制JavaScript（及其超级语言）文件的格式化程序。
- **`    --javascript-formatter-indent-style`**=_`<tab|space>`_ &mdash;
  应用于JavaScript（及其超级语言）文件的缩进样式。
- **`    --javascript-formatter-indent-size`**=_`NUMBER`_ &mdash;
  应用于JavaScript（及其超级语言）文件的缩进大小。默认为2。
- **`    --javascript-formatter-indent-width`**=_`NUMBER`_ &mdash;
  应用于JavaScript（及其超级语言）文件的缩进大小。默认为2。
- **`    --javascript-formatter-line-ending`**=_`<lf|crlf|cr>`_ &mdash;
  应用于JavaScript（及其超级语言）文件的行尾类型。
- **`    --javascript-formatter-line-width`**=_`NUMBER`_ &mdash;
  JavaScript（及其超级语言）文件的一行的最大宽度。默认为80。
- **`    --json-formatter-enabled`**=_`<true|false>`_ &mdash;
  控制JSON（及其超级语言）文件的格式化程序。
- **`    --json-formatter-indent-style`**=_`<tab|space>`_ &mdash;
  应用于JSON（及其超级语言）文件的缩进样式。
- **`    --json-formatter-indent-width`**=_`NUMBER`_ &mdash;
  应用于JSON（及其超级语言）文件的缩进大小。默认为2。
- **`    --json-formatter-indent-size`**=_`NUMBER`_ &mdash;
  应用于JSON（及其超级语言）文件的缩进大小。默认为2。
- **`    --json-formatter-line-ending`**=_`<lf|crlf|cr>`_ &mdash;
  应用于JSON（及其超级语言）文件的行尾类型。
- **`    --json-formatter-line-width`**=_`NUMBER`_ &mdash;
  JSON（及其超级语言）文件的一行的最大宽度。默认为80。

**适用于所有命令的全局选项**

- **`    --colors`**=_`<off|force>`_ &mdash;
  设置标记的格式化模式："off"将所有内容输出为纯文本，"force"无论控制台输出是否兼容，都强制使用ANSI格式化标记
- **`    --use-server`** &mdash;
  连接到正在运行的Biome守护程序服务器。
- **`    --verbose`** &mdash;
  打印有关诊断的额外详细建议
- **`    --config-path`**=_`PATH`_ &mdash;
  设置biome.json配置文件所在的文件系统路径
- **`    --max-diagnostics`**=_`NUMBER`_ &mdash;
  限制显示的诊断数量。
  [默认值：20]
- **`    --skip-errors`** &mdash;
  跳过包含语法错误的文件，而不是发出错误诊断。
- **`    --no-errors-on-unmatched`** &mdash;
  在命令执行期间没有处理任何文件时静音错误。
- **`    --error-on-warnings`** &mdash;
  告诉Biome如果一些诊断发出警告，则以错误代码退出。
- **`    --log-level`**=_`<none|debug|info|warn|error>`_ &mdash;
  日志级别。从最详细到最简洁依次为：debug、info、warn、error。

  值`none`不显示任何日志。

  [默认值：none]

- **`    --log-kind`**=_`<pretty|compact|json>`_ &mdash;
  日志的显示样式。
  [默认值: pretty]
- **`    --diagnostic-level`**=_`<info|warn|error>`_ &mdash;
  要显示的诊断级别。从最低到最重要的顺序是：info、warn、error。传递 `--diagnostic-level=error` 将只打印只包含错误的诊断信息。
  [默认值: info]

**可用的定位参数项:**

- _`PATH`_ &mdash;
  单个文件、单个路径或路径列表

**可用的选项:**

- **`    --apply`** &mdash;
  应用安全修复、格式化
- **`    --apply-unsafe`** &mdash;
  应用安全修复和不安全修复，格式化和导入排序
- **`    --formatter-enabled`**=_`<true|false>`_ &mdash;
  允许启用或禁用格式化检查。
- **`    --linter-enabled`**=_`<true|false>`_ &mdash;
  允许启用或禁用语法检查。
- **`    --organize-imports-enabled`**=_`<true|false>`_ &mdash;
  允许启用或禁用导入排序。
- **`    --stdin-file-path`**=_`PATH`_ &mdash;
  当你想要格式化从 `stdin` 管道输入的代码并将输出打印到 `stdout` 时，使用此选项。

  示例: `echo 'let a;' | biome check --stdin-file-path=file.js`

  打印帮助信息

### `BIOME_BINARY`

```
# Nix 衍生品示例; 二进制文件路径来自 "${pkgs.biome}/bin/biome"

```
